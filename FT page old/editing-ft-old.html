FAQ

React Data Grid Editing Questions

What editing modes are available? +

The React Data Grid supports inline editing, batch editing, dialog editing, and template editing. Choose the mode that best matches your user workflow and data entry requirements.

Can values be validated before saving?
+

Yes. Built-in validation rules can be applied to cells and fields. You can also create custom validation logic to ensure data quality and consistency before records are saved.

Can multiple changes be saved together?
+

Yes. Batch editing supports multiple row edits before saving. All pending changes are saved together in one transaction for coordinated updates.

Can custom editors be used?
+

Yes. Custom editors can be used in any editing mode. Support rich editors, date pickers, dropdowns, and specialized input controls for any data type.

Does editing work with remote data?
+

Yes. Editing integrates with remote data sources. Submit changes to APIs or backends while maintaining synchronization with the grid UI.

Which editing mode is best for complex records?
+

Dialog or template editing is ideal for complex records with many fields or custom layouts. These modes provide more space and flexibility for detailed data entry.
