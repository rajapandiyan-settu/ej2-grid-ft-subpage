Choose the Right Approach

Choose the Selection Experience That Fits Your Workflow

Different use cases need different selection modes. Choose the mode that matches your application's data interaction patterns.

📋

Normal Row Selection

Users click rows to select them. Supporting multiple selections with Ctrl+Click or Shift+Click for range selection.

Best for
General-purpose row selection, data review, and filtering workflows.
Use When
Users need a familiar and simple way to select one or multiple rows without additional UI elements.
☑️

Checkbox Selection

Each row has a checkbox. Header checkbox lets users select all rows at once, making bulk operations intuitive.

Best for
Bulk actions, explicit selection, and accessibility-focused interfaces.
Use When
Users frequently perform actions on multiple records and need clear visual confirmation of selected items.
🔀

Conditional Checkbox Selection

Control which rows can be selected using custom conditions. Only valid rows appear selectable to prevent invalid operations.

Best for
Restricted selections, validation-dependent workflows, and business-rule enforcement.
Use When
Only certain records should be eligible for selection, preventing invalid user actions.
🔲

Cell Selection

Users select individual cells or cell ranges. Precise control for tasks like data entry, analysis, or targeted edits.

Best for
Fine-grained data editing, spreadsheet-like workflows, and precise data highlighting.
Use When
Users need to interact with specific data points rather than whole records.
📊

Column Selection

Select entire columns by clicking headers or using keyboard shortcuts. Useful for operations affecting all rows in a column.

Best for
Column-level operations, data export, formatting, and analysis workflows.
Use When
Users need to analyze, format, copy, or apply actions to all values within a column.

FAQ

React Data Grid Selection Questions

+ What selection modes are available?
The Syncfusion React Data Grid supports Row Selection (normal and checkbox), Cell Selection, and Column Selection. Each mode can be combined with keyboard shortcuts and conditional logic to match your workflow needs.
+ Can I customize the selection behavior?
Yes. The grid supports custom selection handlers, conditional selection rules, keyboard shortcut configuration, and styling. You can define which rows/cells can be selected and customize the visual feedback.
+ Does selection work with remote or dynamic data?
Yes, selection works seamlessly with remote data sources. The grid maintains selection state across pagination and sorting operations. Use the selection API to get selected rows at any time, regardless of data source.
+ Can I perform bulk actions on selected rows?
Absolutely. Use the selection API to get all selected rows, then perform batch operations like update, delete, export, or custom actions. The grid provides callbacks to capture selection changes for real-time action triggers.
+ Which selection mode should I use?
Choose based on your use case: Row Selection for general workflows, Checkbox Selection for explicit/bulk actions, Cell Selection for precise editing, Column Selection for column-level operations, and Conditional Selection when business rules restrict which items can be selected.
+ Can I use Shift+Click and Ctrl+Click for range and multi-selection?
Yes, the grid supports standard keyboard shortcuts out of the box. Shift+Click selects a range of rows, Ctrl+Click (or Cmd+Click on Mac) toggles individual row selection. These shortcuts work seamlessly with all selection modes and can be customized as needed.
+ How can I restrict selection based on business rules?
Use Conditional Checkbox Selection to apply custom logic that determines which rows are selectable. You can define selection rules based on row data, status, permissions, or any other criteria. Implement the conditional handler to enable/disable selection dynamically.
+ Can I get the selected data programmatically?
Yes, the grid exposes a comprehensive selection API. You can retrieve selected rows, cells, or columns at any time using methods like `getSelectedRowIndexes()`, `getSelectedRowsData()`, and `getSelectedCellData()`. These methods make it easy to perform actions on selections.
+ Does selection persist when the grid is filtered or sorted?
Yes, selections are maintained across filtering and sorting operations by default. The grid tracks selections by row identifier (primary key) rather than row index, ensuring selections remain consistent even when the data order changes. You can configure this behavior to suit your needs.
