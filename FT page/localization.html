FAQ

React Data Grid Localization Questions

What languages and locales does the React Data Grid support? ▼

The React Data Grid supports over 80+ locales including major languages like English, Spanish, French, German, Chinese, Japanese, Arabic, and more. You can also create custom locale files for specific regional requirements or organizational terminology.

Can I change the grid locale dynamically without reloading the page? ▼

Yes, you can switch the grid locale at runtime by updating the locale property. The grid automatically refreshes to apply the new language, date formats, number formats, and UI text without requiring a page reload.

Does localization work with remote data sources? ▼

Yes, localization applies to all data sources including remote APIs. The grid handles client-side formatting for dates, numbers, and currency regardless of the data source. You can also configure server-side formatting to match the selected locale.

How do I add RTL (Right-to-Left) support for Arabic or Hebrew? ▼

Enable RTL mode by setting the enableRtl property to true and loading the appropriate locale file. The grid automatically reverses the layout, text direction, and column order to provide native RTL language support.

Can I customize specific UI text without changing the entire locale? ▼

Yes, you can override specific text strings using the L10n.load() method. This allows you to customize toolbar buttons, pager text, filter menus, and validation messages while keeping the rest of the locale intact.

Where can I find locale files for the React Data Grid? ▼

Locale files are available in the @syncfusion/ej2-locale package. You can import the required locale JSON file and register it using the L10n.load() method. Custom locale files can also be created following the standard locale structure.
