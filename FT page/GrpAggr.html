Why Grouping & Aggregates Matter

Turn Large Datasets into Meaningful Summaries

Flat grids become difficult to analyze as data grows. Users often need category breakdowns, regional totals, and summary information without manually reviewing thousands of rows.

Grouping and aggregates help transform raw data into organized, actionable insights, making analysis faster and more efficient.

Choose the Right Grouping Strategy

Choose the Right Grouping and Aggregation Strategy

Different applications require different ways to organize and summarize data. Choose the grouping and aggregation approach that best fits your reporting, analytics, and business workflow requirements.

🖱️

Drag-and-Drop Grouping

Allow users to organize records dynamically by dragging columns into the grouping area during runtime.

Best for: Dashboards, sales reports, ad-hoc exploration.
Use when users need to change the grouping themselves at runtime.
⚙️

Programmatic Grouping

Set group columns in configuration so the grid always loads in a consistent, pre-defined state.

Best for: Fixed reports, admin panels, order management screens.
Use when the grouping structure must stay consistent for all users.
∑

Built-in Aggregate Functions

Apply Sum, Average, Count, Min, or Max to any numeric column. Results appear at the group footer and grid footer.

Best for: Financial summaries, inventory totals, KPI dashboards.
Use when standard statistical summaries meet your reporting needs.
🔧

Custom Aggregate Functions

Write a function that receives group data and returns any value — weighted averages, margin ratios, or business KPIs.

Best for: Finance apps, BI screens, custom analytics tools.
Use when built-in functions don't cover your calculation.
🌐

Server-Side Grouping

Group parameters are sent with each data request. The backend handles grouping and aggregation; the grid renders the result.

Best for: Large datasets, data warehouses, remote API sources.
Use when loading all rows into the browser is not feasible.
📄

Caption Templates

Replace the default group header with a custom template — add icons, badges, or inline aggregate values per group row.

Best for: Branded apps, multi-tenant portals, styled reports.
Use when the default caption layout doesn't fit your design.
📅

Group by Formatted Values

Organize records using formatted date, time, or numeric values instead of raw data fields for more meaningful analysis.

Best for: Sales reports, financial dashboards, and time-based analytics.
Use when data needs to be grouped by month, year, quarter, or custom formats.
📑

Grouping with Paging

Combine grouped records with paging to improve navigation and maintain performance across large result sets.

Best for: Enterprise applications and high-volume data grids.
Use when grouped datasets are too large to display on a single page.

FAQ

Frequently Asked Questions About Grouping and Aggregates

▼ Can users group records without writing code?

Yes. Enable the grouping panel and users can drag and drop column headers to group records directly within the grid without writing additional code.

▼ Can aggregate values be displayed in group captions?

Yes. Aggregate values can be displayed directly in group caption rows, allowing users to see summary information without expanding groups.

▼ Can multiple aggregate functions be applied to a single column?

Yes. A column can display multiple aggregate calculations such as Sum, Average, Count, Min, and Max simultaneously.

▼ Can grouped columns remain visible after grouping?

Yes. You can configure the grid to either display or hide grouped columns based on your application requirements.

▼ Can users reorder grouped columns?

Yes. Grouped columns can be reordered to change the hierarchy and structure of grouped records dynamically.

▼ Can data be grouped using formatted values such as months or years?

Yes. The grid supports grouping data using formatted values, helping users organize records by dates, periods, and custom formats.

▼ Can grouped records be expanded and collapsed programmatically?

Yes. Developers can control group expansion and collapse behavior through code to create customized user experiences.

▼ Can multiple columns be grouped simultaneously?

Yes. Multiple columns can be grouped to create hierarchical views, making large datasets easier to browse and analyze.

▼ Which aggregate functions are supported?

The Data Grid supports Sum, Average, Count, Min, and Max aggregations. These values can be displayed at both group and grid footer levels.

▼ Can custom aggregate calculations be implemented?

Yes. Developers can create custom aggregate functions to calculate application-specific metrics, ratios, KPIs, or weighted values.

▼ Does grouping support remote and large datasets?

Yes. Grouping and aggregation can be performed using server-side processing to efficiently handle large datasets.

▼ When should I use programmatic grouping instead of drag-and-drop grouping?

Use programmatic grouping when a predefined grouping structure is required. Use drag-and-drop grouping when users need flexibility to reorganize data at runtime.
