Why Performance Optimization Matters

Deliver Fast, Responsive User Experiences

Modern applications demand instant responsiveness and smooth interactions. Choose the Right Performance Strategy

DOM Virtualization with Virtual Scroll

Renders rows dynamically based on configurable buffer values. By adjusting the buffer, you control how many additional rows are rendered above and below the visible viewport, optimizing the balance between smooth scrolling and minimal DOM elements.

Best for: Applications needing fine-tuned control over rendering performance through customizable buffer settings.

Use when you want to optimize rendering performance with adjustable buffer values for your specific use case.

Virtual Scrolling

Renders rows based on the configured page size setting. The number of rows rendered in the viewport is determined by the page size value, providing predictable rendering behavior and consistent performance with a fixed rendering window.

Best for: Applications requiring consistent rendering with a defined number of visible rows for predictable performance.

Use when you want controlled, page-size-based rendering for smooth scrolling with predictable resource usage.

Paging

Divides data into discrete pages with a fixed number of records per page. Users navigate through pages using pagination controls. Provides excellent performance by loading and rendering only one page at a time, while offering familiar navigation patterns.

Best for: Applications where users process data in chunks, need predictable loading behavior, or prefer traditional navigation.

Use when you want fast initial loads, controlled resource usage, and familiar pagination controls.

Infinite Scrolling

Automatically loads and appends the next set of records when users scroll to the bottom. Provides seamless browsing experience similar to social media feeds while progressively loading data and maintaining optimal performance.

Best for: Mobile-friendly applications and modern interfaces where users explore data continuously without explicit load actions.

Use when building consumer-facing apps or when you want to provide fluid exploration experiences with automatic loading.

Frequently Asked Questions

Can virtual scrolling work with remote data sources?
Yes, virtual scrolling fully supports remote data binding. The grid efficiently loads data in chunks from your server as users scroll, minimizing initial load time and bandwidth usage.
How does virtualization improve grid performance?
Virtualization renders only visible rows and columns, dramatically reducing DOM elements and memory usage. This results in faster rendering, smoother scrolling, and better responsiveness regardless of data volume.
Can I combine paging with other grid features like filtering and sorting?
Absolutely. All performance strategies work seamlessly with filtering, sorting, grouping, and other grid features. The grid automatically recalculates and maintains performance regardless of applied operations.
Which performance strategy provides the best user experience?
It depends on your use case. Virtual scrolling offers smooth continuous navigation, paging provides predictable chunks, DOM virtualization delivers maximum rendering speed, and infinite scrolling creates seamless exploration. Choose based on your application's workflow needs.
Is it possible to switch between performance strategies dynamically?
Yes, you can enable or disable these features programmatically at runtime. This allows you to switch strategies based on user preferences or data size.
Where can I find complete implementation examples?
Check the documentation links in the Implementation Resources section above. Each includes working code samples, configuration options, and best practices specific to each performance strategy.
